AI-Powered Lead Qualification System
Demo Video
Problem Solved

Business owners manually sorting through form submissions, guessing which leads are worth following up on. Hours wasted. Hot leads going cold.

Automation Flow
01
Google Form receives lead submission with name, business type, need, budget, and email
02
Apps Script webhook bridge fires instantly and sends data to Make.com
03
Google Sheets logs the raw lead data immediately as the first action
04
OpenRouter AI (LLM) scores the lead 1–10 and returns JSON with score, tier, reason, and suggested action
05
JSON Parser extracts score and tier as individual mappable fields
06
Google Sheets updated with AI score and tier in the same row
07
Router (3 paths) — Hot (≥7): Gmail alert + Telegram. Warm (4–6): auto-reply email to lead. Cold (≤3): sheet only.
08
Error handler on HTTP module sends Gmail alert with error details if AI call fails

The Technical Problem

Google Forms has no native webhook support. Most solutions require expensive third-party tools. This implementation uses Google Apps Script to bridge the gap at zero cost.

How It Works
01
Apps Script trigger attached to the form's onFormSubmit event
02
Response data extracted by index position from the form response object
03
JSON.stringify(data) automatically sanitizes all special characters before transmission
04
UrlFetchApp.fetch() fires a POST request to the Make.com webhook URL instantly
05
Make.com receives clean JSON with all form fields as mappable variables
Why This Matters

Replaces the need for paid form tools like Typeform or Tally. Any client already using Google Forms can connect to any Make.com automation instantly — no new subscriptions required.
